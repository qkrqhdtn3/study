codeforces/contests/div2/1002
A.cpp
  문제 설명
길이가 n인 배열 a, b가 주어지고, 이 배열들은 원소인 정수 x가 있다면, x는 최소 2번 이상 나오는 것이 보장되어 있다.
배열 a를 재배치해서 c[i] = a[i] + b[i]인 c 배열을 만든다고 할 때,
c가 최소한 3개의 다른 정수를 원소로 가지는지 판별하는 문제이다. 이 조건이 맞다면 YES 틀리면 NO이다.

  풀이 아이디어
  경우의 수
1. a가 최소한 3개의 다른 정수를 원소로 가질 때
b가 1개의 다른 정수만을 원소로 가져도 c는 최소한 3개의 다른 정수를 원소로 가진다.
2. b가 최소한 3개의 다른 정수를 원소로 가질 때
a가 1개의 다른 정수만을 원소로 가져도 c는 최소한 3개의 다른 정수를 원소로 가진다.
따라서 어느 한쪽이 최소한 3개의 다른 정수를 원소로 가지면 답은 YES다.
3. a가 2개의 다른 정수를 원소로 가질 때
a가 1 1 2 2 라면
3-1. b가 1 1 1 1인 경우 답은 NO다.
어떠한 경우를 생각해보더라도 1개의 다른 정수를 원소로 가지면 답은 NO다.
3-2. b가 1 2 1 2인 경우 답은 YES다.
b가 2개의 다른 정수를 가진 경우 답이 YES가 될 수 있다.

  3-2의 경우
문제의 조건에 따라서
a가 다른 정수 2쌍을 각각 최소 2개 가지고 있고,
b가 다른 정수 2쌍을 각각 최소 2개 가지고 있다.
이 경우 조건을 만족하는 가장 작은 배열 두개는 밑과 같다.
a = {a1, a1, a2, a2}
b = {b1, b1, b2, b2}
c 배열은 총 네개의 정수를 만들 수 있다.
c = {
a1 + b1,
a1 + b2,
a2 + b1,
a2 + b2}
만약 a1 < a2, b1 < b2 라면
a1 + b2 == a2 + b1일 수도 있다. 따라서
a1 + b1 < a1 + b2 <= a2 + b1 < a2 + b2 이거나
a1 + b1 < a2 + b1 <= a1 + b2 < a2 + b2 이다.
따라서 문제의 조건에 맞는 배열 a와 b는 둘 다 각각 2개의 다른 정수를 가지는 경우는
최소한 3개의 정수를 만들 수 있고 답은 YES다.

정리를 해보면
답이 YES인 경우는
a가 다른 정수 3개 이상을 가지면 b는 1개 이상의 다른 정수를 가지면 된다.
a가 다른 정수 2개를 가지면 b는 2개 이상의 다른 정수를 가지면 된다.
a가 다른 정수 1개를 가지면 b는 3개 이상의 다른 정수를 가지면 된다.
이를 구현하면 된다.
